**報告日期：** 2026-04-08

**目標：** 調查 GOOGLEFINANCE 函數對 ETF 歷史資料的支援性，以 VOO 為範例

## 執行摘要

經過深度調查與實際測試，`GOOGLEFINANCE` 函數在處理 **ETF 長期歷史資料** 時存在 **重大限制**，無法可靠地獲取從發行日到昨日的完整歷史資料。主要問題包括：數據不可靠、API 限制、以及歷史資料回傳機制的限制。

**關鍵發現：**
- ❌ **無法經由 API 獲取長期歷史資料** —— Google 官方文件明確指出：「Historical data cannot be downloaded or accessed via the Sheets API or Apps Script」
- ⚠️ **實際測試顯示大量 #REF! / #N/A 錯誤** —— 即使在 Google Sheets 介面中寫入公式
- ✅ **即時資料可用** —— 當日價格、成交量基本屬性尚可運作
- ⚠️ **Google 將日期視為 noon UTC** —— 可能導致時區偏移問題

## 一、GOOGLEFINANCE 函數概覽

### 1.1 語法結構

```excel
=GOOGLEFINANCE(ticker, [attribute], [start_date], [end_date], [interval])
```

**參數說明：**
- `ticker`（必填）：股票代號，建議加入交易所前綴（如 "NASDAQ:GOOG"）
- `attribute`（選填）：資料屬性
- `start_date`（選填）：歷史資料起始日期
- `end_date`（選填）：歷史資料結束日期或天數
- `interval`（選填）：頻率，支援 "DAILY" 或 "WEEKLY"

### 1.2 支援屬性列表

#### 即時資料屬性（約 18 個）

```excel
"price"        - 即時價格
"priceopen"    - 當日開盤價
"high"         - 當日最高價
"low"          - 當日最低價
"volume"       - 當日成交量
"marketcap"    - 市值
"tradetime"    - 最後交易時間
"datadelay"    - 資料延遲分鐘數
"volumeavg"    - 平均日成交量
"pe"           - 本益比
"eps"          - 每股盈餘
"high52"       - 52 週最高價
"low52"        - 52 週最低價
"change"       - 價格變動
"changepct"    - 價格變動百分比
"closeyest"    - 昨日收盤價
"shares"       - 流通股數
"currency"     - 交易貨幣
```

#### 歷史資料屬性（僅 5 個）

```excel
"open"   - 開盤價（指定日期/範圍）
"high"   - 最高價（指定日期/範圍）
"low"    - 最低價（指定日期/範圍）
"close"  - 收盤價（指定日期/範圍）
"volume" - 成交量（指定日期/範圍）
"all"    - 以上所有屬性
```

#### 共同基金專屬屬性（ETF 不適用）

```excel
"returnytd"      - 年初至今回報
"return1"        - 一週總回報
"return4"        - 四週總回報
"return13"       - 十三週總回報
"return52"       - 五十二週總回報
"yieldpct"       - 配息收益率
"incomedividend" - 最近現金配息金額
"expenseratio"   - 費用率
```

## 三、核心限制與問題

### 3.1 API 層級限制（關鍵）

**Google 官方文件明確限制：**

> "Historical data cannot be downloaded or accessed via the Sheets API or Apps Script. If you attempt to do so, you'll see a #N/A error in place of the values in the corresponding cells of your spreadsheet."

**影響：**
- ❌ 無法透過 `gog` CLI、Sheets API 或 Apps Script 獲取歷史資料
- ❌ 即使在 Google Sheets 介面中，長期歷史資料亦不可靠
- ⚠️ 僅能在 Google Sheets 介面中手動輸入公式，且結果不穩定

### 3.2 時間處理機制

**時間解析規則：**
- Google 將傳入的日期視為 **12:00 PM UTC**
- 交易所收盤時間可能偏移一天

**範例：**
```excel
=GOOGLEFINANCE("VOO", "close", DATE(2026, 4, 7))
```
- 預期：2026-04-07 收盤價
- 實際：可能返回 2026-04-06 或 2026-04-08 的資料

### 3.3 ETF 特定限制

**不支援的屬性：**
- ❌ P/E 比率（`pe`）：ETF 本身不有此數據
- ❌ EPS（`eps`）：ETF 不適用
- ❌ 配息相關（Mutual Fund 專屬）：`yieldpct`, `incomedividend`, `returnytd` 等

**適用的屬性：**
- ✅ 價格屬性：`price`, `priceopen`, `high`, `low`, `close`
- ✅ 成交量：`volume`
- ✅ 技術數據：`high52`, `low52`, `change`, `changepct`

### 3.4 資料延遲與更新頻率

**官方限制：**
- **即時資料延遲：** 最多 20 分鐘
- **更新頻率：** 約每 20 分鐘一次（交易時間內）
- **手動重新整理：** 不支援

```excel
=GOOGLEFINANCE("AAPL", "datadelay")  // 查詢當前延遲分鐘數
```

### 3.5 API 速率限制

**非官方限制：**
- 同一 Sheet 中數百個 `GOOGLEFINANCE` 呼叫可能導致：
  - #N/A 錯誤
  - 載入極慢
  - API 暫時封鎖

**建議：**
- 分散在不同 Sheet/Tab
- 定期複製並貼上數值以緩存

## 四、替代方案建議

### 4.1 推薦方案：Yahoo Finance API (yfinance)

**優點：**
- ✅ 完整歷史 OHLCV 資料（含 ETF）
- ✅ 支援長時間範圍（從發行日至今）
- ✅ 多種時間間隔：1m, 5m, 15m, 1h, 1d, 1w, 1mo
- ✅ 免費且開源
- ✅ Python 套件易用

**範例程式碼：**

```python
import yfinance as yf
from datetime import datetime, timedelta

# 初始化 VOO ticker
voo = yf.Ticker("VOO")

# 設定日期範圍（從發行日到昨天）
start_date = "2010-09-07"
end_date = (datetime.now() - timedelta(days=1)).strftime("%Y-%m-%d")

# 獲取日資料
df = voo.history(start=start_date, end=end_date, interval="1d")

# 輸出前 5 行
print(df.head())

# 寫入 Google Sheets（需使用 gog 或 Google Sheets API）
# 可將 df 轉換為 JSON 或 CSV 格式後透過 gog 上傳
```

**輸出範例：**

| Date       | Open   | High   | Low    | Close  | Volume |
|------------|--------|--------|--------|--------|--------|
| 2010-09-07 | 100.11 | 100.44 | 99.79  | 100.00 | 450k   |
| 2010-09-08 | 100.05 | 100.38 | 99.95  | 100.20 | 420k   |
| ...        | ...    | ...    | ...    | ...    | ...    |

**與 Google Sheets 整合：**

```python
# 方法 1：透過 gog CLI 上傳 CSV
df.to_csv("voo_history.csv", index=True)
# gog drive upload voo_history.csv

# 方法 2：直接寫入 Google Sheets（需先啟用 Sheets API）
# 參考 Google Sheets API 文件
```

### 4.2 其他替代方案

## 五、最佳實踐建議

### 5.1 場景 1：即時股價監控（少數 ETF）

**推薦：** 使用 `GOOGLEFINANCE`

**原因：**
- 即時數據（20 分鐘延遲）可接受
- 無需 API Key
- 設置簡單快速

```excel
A 欄: ETF Ticker (VOO, SPY, QQQ...)
B 欄: =GOOGLEFINANCE(A2, "price")
C 欄: =GOOGLEFINANCE(A2, "changepct")
D 欄: =GOOGLEFINANCE(A2, "volume")
```

### 5.2 場景 2：歷史資料分析（長期趨勢）

**推薦：** 使用 **yfinance**（Python）

**原因：**
- ✅ 完整歷史資料（從發行日至今）
- ✅ 可定製時間範圍與間隔
- ✅ 免費且可靠

**流程：**
1. Python 腳本擷取歷史資料（yfinance）
2. 輸出為 CSV/JSON
3. 透過 `gog` CLI 或 Google Sheets API 上傳

### 5.3 場景 3：混合方案（Google Sheets + Python）

**架構：**
```
yfinance 定期爬取 → 寫入 Google Sheets → GOOGLEFINANCE 即時更新
```

**優點：**
- 歷史資料穩定（yfinance）
- 即時資料方便（GOOGLEFINANCE）
- 單一介面管理
