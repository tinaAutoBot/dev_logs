## 一、GOOGLEFINANCE 函數概述

`GOOGLEFINANCE` 是 Google Sheets 內建的金融資料函數，可從 Google Finance 獲取即時或歷史金融資料。

### 基本語法

```
=GOOGLEFINANCE(ticker, [attribute], [start_date], [end_date|num_days], [interval])
```

### 參數說明

| 參數 | 必填 | 說明 |
|------|------|------|
| `ticker` | ✅ | 股票/ETF 代碼，建議加上交易所前綴（如 `NYSEARCA:VOO`） |
| `attribute` | ❌ | 要取得的資料屬性，預設為 `price` |
| `start_date` | ❌ | 歷史資料起始日期 |
| `end_date` | ❌ | 歷史資料結束日期（或使用 `num_days` 指定天數） |
| `interval` | ❌ | 資料頻率：`DAILY`（每日）或 `WEEKLY`（每週） |

## 二、可取得的資料屬性

### 2.1 即時資料屬性（Real-time Attributes）

| 屬性 | 說明 | ETF 支援 |
|------|------|----------|
| `price` | 即時價格（延遲約 20 分鐘） | ✅ |
| `priceopen` | 當日開盤價 | ✅ |
| `high` | 當日最高價 | ✅ |
| `low` | 當日最低價 | ✅ |
| `volume` | 當日交易量 | ✅ |
| `marketcap` | 市值 | ⚠️ 部分支援 |
| `tradetime` | 最後交易時間 | ✅ |
| `datadelay` | 資料延遲時間（分鐘） | ✅ |
| `volumeavg` | 平均每日交易量 | ✅ |
| `pe` | 本益比（P/E ratio） | ❌ ETF 不適用 |
| `eps` | 每股盈餘 | ❌ ETF 不適用 |
| `high52` | 52 週最高價 | ✅ |
| `low52` | 52 週最低價 | ✅ |
| `change` | 價格變動 | ✅ |
| `changepct` | 價格變動百分比 | ✅ |
| `closeyest` | 昨日收盤價 | ✅ |
| `shares` | 流通股數 | ⚠️ 部分支援 |
| `currency` | 交易幣別 | ✅ |
| `beta` | Beta 值 | ⚠️ 部分支援 |

### 2.2 歷史資料屬性（Historical Attributes）

**⚠️ 重要：** 當指定日期參數時，只能使用歷史資料屬性。

| 屬性 | 說明 | 回傳格式 |
|------|------|----------|
| `open` | 開盤價 | 日期 + 數值 |
| `close` | 收盤價 | 日期 + 數值 |
| `high` | 當日最高價 | 日期 + 數值 |
| `low` | 當日最低價 | 日期 + 數值 |
| `volume` | 交易量 | 日期 + 數值 |
| `all` | 以上所有資料 | 日期 + OHLC + Volume |

### 2.3 共同基金專用屬性（Mutual Fund Attributes）

**⚠️ 這些屬性僅適用於共同基金，不支援 ETF！**

| 屬性 | 說明 |
|------|------|
| `returnytd` | 年初至今報酬率 |
| `return1` | 一週總報酬率 |
| `return4` | 四週總報酬率 |
| `return13` | 13 週總報酬率 |
| `return52` | 52 週（年度）總報酬率 |
| `return156` | 156 週（3 年）總報酬率 |
| `return260` | 260 週（5 年）總報酬率 |
| `yieldpct` | 配息殖利率 |
| `incomedividend` | 最近現金股利金額 |
| `incomedividenddate` | 最近現金股利日期 |
| `morningstarrating` | 晨星評級 |
| `expenseratio` | 管理費用率 |

## 三、ETF 歷史資料實作範例（以 VOO 為例）

### 3.1 VOO 基本資訊

- **名稱：** Vanguard S&P 500 ETF
- **交易所：** NYSEARCA
- **發行日期：** 2010-09-07
- **追蹤指數：** S&P 500 Index

### 3.2 取得即時價格

```google sheets
// 基本寫法
=GOOGLEFINANCE("VOO")

// 建議加上交易所前綴
=GOOGLEFINANCE("NYSEARCA:VOO")

// 取得特定屬性
=GOOGLEFINANCE("NYSEARCA:VOO", "price")
=GOOGLEFINANCE("NYSEARCA:VOO", "volume")
```

### 3.3 取得歷史資料

```google sheets
// 取得特定日期的收盤價
=GOOGLEFINANCE("NYSEARCA:VOO", "close", DATE(2024,1,15))

// 取得日期範圍的收盤價（每日）
=GOOGLEFINANCE("NYSEARCA:VOO", "close", DATE(2024,1,1), DATE(2024,1,31), "DAILY")

// 取得過去 30 天的收盤價
=GOOGLEFINANCE("NYSEARCA:VOO", "close", TODAY()-30, TODAY())

// 取得過去 5 年的週收盤價
=GOOGLEFINANCE("NYSEARCA:VOO", "close", TODAY()-365*5, TODAY(), "WEEKLY")

// 取得所有 OHLCV 資料
=GOOGLEFINANCE("NYSEARCA:VOO", "all", DATE(2024,1,1), DATE(2024,12,31))
```

### 3.4 從發行日取得完整歷史資料

```google sheets
// VOO 發行於 2010-09-07，取得至今的全部歷史資料
=GOOGLEFINANCE("NYSEARCA:VOO", "all", DATE(2010,9,7), TODAY(), "DAILY")

// 以週資料減少資料量
=GOOGLEFINANCE("NYSEARCA:VOO", "all", DATE(2010,9,7), TODAY(), "WEEKLY")
```

## 四、重要限制與注意事項

### 4.1 資料延遲

- **即時資料延遲約 20 分鐘**
- 不適合用於日內交易或需要即時報價的場景

### 4.2 歷史資料限制

| 限制 | 說明 |
|------|------|
| 最小時間粒度 | 僅支援日（DAILY）或週（WEEKLY），**不支援分鐘或小時資料** |
| num_days 上限 | 使用 `num_days` 參數時，最大值為 **50** |
| 透過 API 取得 | ❌ 不支援透過 Sheets API 或 Apps Script 取得歷史資料 |
| 資料完整性 | 部分日期可能缺值（假日、休市日等） |

### 4.3 ETF 專用限制

#### ❌ 無法取得的資料

| 資料類型 | 說明 |
|----------|------|
| 股利殖利率 | `DIVIDENDYIELD` 不存在 |
| 股利歷史 | 無法取得配息記錄 |
| 每股股利 | `DIVIDENDPERSHARE` 不存在 |
| 報酬率計算 | `return1`, `return52` 等**僅適用於共同基金** |
| 內扣費用 | ETF 的 expense ratio 需另外查詢 |
| 淨值(NAV) | 需透過其他來源取得 |

#### ⚠️ ETF vs 共同基金屬性差異

```google sheets
// ❌ 這個公式對 ETF 會傳回 #N/A 錯誤
=GOOGLEFINANCE("VOO", "return52")  // ETF 不支援

// ✅ 正確做法：使用共同基金代碼
=GOOGLEFINANCE("VFIAX", "return52")  // 共同基金適用
```

### 4.4 速率限制

- Google 未公開明確的 API 呼叫限制
- 大量的 `GOOGLEFINANCE` 公式可能導致：
  - 試算表載入變慢
  - `#N/A` 錯誤
  - 資料暫時無法取得

### 4.5 交易所支援

- 主要支援美國交易所
- 國際股票需加上交易所前綴（如 `LON:VOD`, `TYO:7203`）
- 部分小型交易所或 OTC 股票可能無資料

## 五、實用進階技巧

### 5.1 使用 INDEX 取得單一數值

```google sheets
// 預設會回傳陣列（含標題列）
// 使用 INDEX 取得特定儲存格的值
=INDEX(GOOGLEFINANCE("VOO", "close", TODAY()-1), 2, 2)
```

### 5.2 結合 QUERY 篩選資料

```google sheets
// 找出收盤價高於某個價位的日期
=QUERY(GOOGLEFINANCE("VOO", "close", DATE(2024,1,1), TODAY()), 
       "SELECT * WHERE Col2 > 400")
```

### 5.3 使用 SPARKLINE 繪製迷你圖

```google sheets
// 在單一儲存格內繪製 30 天價格走勢
=SPARKLINE(GOOGLEFINANCE("VOO", "close", TODAY()-30, TODAY()))
```

### 5.4 計算簡易移動平均

```google sheets
// 計算 50 日移動平均線（近似值）
=AVERAGE(INDEX(GOOGLEFINANCE("VOO", "close", TODAY()-55, TODAY()), 2, 2, 50, 1))
```

### 5.5 錯誤處理

```google sheets
// 使用 IFERROR 避免錯誤訊息
=IFERROR(GOOGLEFINANCE("VOO", "price"), "資料暫時無法取得")
```

## 六、收集 ETF 歷史資料的建議流程

### 6.1 方案一：直接使用 GOOGLEFINANCE

**優點：**
- 免費、無需 API key
- 即時更新
- 整合於 Google Sheets

**缺點：**
- 無法取得股利資料
- 無法計算報酬率
- 資料延遲 20 分鐘
- 歷史資料可能不完整

### 6.2 方案二：搭配其他資料來源

| 資料類型 | 建議來源 |
|----------|----------|
| 價格/成交量 | GOOGLEFINANCE |
| 股利歷史 | Yahoo Finance、ETF 官網 |
| 報酬率計算 | 自行用公式計算或使用 Morningstar |
| 淨值(NAV) | ETF 發行商官網（如 Vanguard） |

### 6.3 建議的資料收集架構

```
Google Sheet 工作表結構建議：

Sheet 1: ETF_List（ETF 清單）
| Ticker | Name | Exchange | Inception Date |
|--------|------|----------|----------------|
| VOO    | Vanguard S&P 500 ETF | NYSEARCA | 2010-09-07 |
| VTI    | Vanguard Total Stock Market ETF | NYSEARCA | 2001-05-24 |
| ...

Sheet 2: Price_Data（價格資料）
使用 IMPORTRANGE 或直接引用 GOOGLEFINANCE 動態更新

Sheet 3: Dividend_Data（股利資料）
手動維護或從其他來源匯入

Sheet 4: Analysis（分析看板）
計算報酬率、風險指標等
```

## 七、以 VOO 為例的完整範例試算表公式

### 7.1 建立 ETF 監控表

```google sheets
// A 欄：Ticker
// B 欄：名稱（手動輸入）
// C 欄：即時價格
=GOOGLEFINANCE(A2, "price")

// D 欄：今日漲跌
=GOOGLEFINANCE(A2, "change")

// E 欄：漲跌幅 %
=GOOGLEFINANCE(A2, "changepct")

// F 欄：成交量
=GOOGLEFINANCE(A2, "volume")

// G 欄：52 週高點
=GOOGLEFINANCE(A2, "high52")

// H 欄：52 週低點
=GOOGLEFINANCE(A2, "low52")

// I 欄：昨日收盤
=GOOGLEFINANCE(A2, "closeyest")
```

### 7.2 取得昨天到發行日的歷史資料

```google sheets
// 以 VOO 為例（發行日：2010-09-07）
// 昨天到發行日的完整歷史資料

=GOOGLEFINANCE("NYSEARCA:VOO", "all", DATE(2010,9,7), TODAY()-1, "DAILY")

// 如果資料量太大，改用週資料
=GOOGLEFINANCE("NYSEARCA:VOO", "all", DATE(2010,9,7), TODAY()-1, "WEEKLY")
```

### 7.3 計算投資報酬率（需自行計算）

```google sheets
// 總報酬率 = (期末價格 - 期初價格) / 期初價格
// 注意：不含股利，單純價格報酬

// 取得 1 年前收盤價
=INDEX(GOOGLEFINANCE("VOO", "close", TODAY()-365), 2, 2)

// 計算 1 年報酬率
=(GOOGLEFINANCE("VOO", "price") - INDEX(GOOGLEFINANCE("VOO", "close", TODAY()-365), 2, 2)) 
 / INDEX(GOOGLEFINANCE("VOO", "close", TODAY()-365), 2, 2)
```

## 八、總結

### GOOGLEFINANCE 對 ETF 的支援摘要

| 資料類型 | 支援度 | 備註 |
|----------|--------|------|
| 即時價格 | ✅ 完整支援 | 延遲約 20 分鐘 |
| 歷史價格（OHLC） | ✅ 完整支援 | 日/週頻率 |
| 交易量 | ✅ 完整支援 | 歷史與即時皆有 |
| 52 週高低點 | ✅ 支援 | 即時屬性 |
| 市值 | ⚠️ 部分支援 | 可能不準確 |
| 本益比/每股盈餘 | ❌ 不適用 | ETF 非股票 |
| 股利資料 | ❌ 不支援 | 需其他來源 |
| 報酬率（return1 等） | ❌ 僅共同基金 | ETF 會傳回 #N/A |
| 費用率 | ❌ 不支援 | 需查 ETF 官網 |
| 淨值(NAV) | ❌ 不支援 | 需查發行商官網 |

### 建議

1. **價格與成交量資料** → 使用 GOOGLEFINANCE（免費、方便）
2. **股利與報酬率資料** → 搭配 Yahoo Finance 或 Morningstar
3. **詳細 ETF 資訊** → 查詢發行商官網（Vanguard、iShares 等）
4. **大量歷史資料** → 考慮使用付費 API（如 MarketXLS）或 Python + yfinance
